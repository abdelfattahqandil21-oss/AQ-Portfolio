# Debugging

1. Reproduce with smallest possible input before touching code
2. State hypothesis clearly before making any change
3. Change one thing at a time and verify
4. Check types, nullability, and async boundaries first
5. Read error messages fully — don't skip the stack trace
6. Add structured logs with context before guessing
7. Confirm fix addresses root cause, not symptom
8. Clean up all debug logs before finishing
